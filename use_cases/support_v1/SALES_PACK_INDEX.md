# Support V1 Sales Pack Index

## What this adds

This index adds one short, practical map of the existing `support_v1` artifacts to use in a commercial or pilot sales conversation so the right document can be pulled quickly without scanning the full folder.

## Decision / credibility docs

| File | Purpose | Use when |
| --- | --- | --- |
| `commercial_pilot_one_pager.md` | Short buyer-facing overview of the problem, current build, pilot shape, and limits. | Use in the first intro call, qualification follow-up, or initial commercial email. |
| `pilot_readiness_decision_brief.md` | Direct readiness position on whether `support_v1` is ready for first pilot outreach now. | Use when a sponsor asks whether the project is actually ready for a bounded pilot. |
| `artifacts/support_v1_readiness_memo.md` | Compact evidence summary across ingest coverage, validation status, and evaluation results. | Use when the buyer wants a tighter proof layer behind the one-pager. |
| `executive_status_brief.md` | Executive snapshot of what is built, what is proven, and what is still missing. | Use when an executive sponsor needs a fast status read before approving next steps. |
| `investor_value_brief.md` | Commercial value framing for why calibrated routing matters beyond workflow detail. | Use when the conversation shifts from pilot mechanics to broader business upside. |

## Commercial docs

| File | Purpose | Use when |
| --- | --- | --- |
| `support_v1_pilot_pricing_framework.md` | Practical pricing and scope logic for lightweight, standard, and extended pilot shapes. | Use when the conversation moves from interest to pilot packaging and commercial terms. |
| `support_v1_roi_model.md` | Conservative ROI model for turning pilot results into buyer value language. | Use when the buyer wants a value case tied to queue assumptions rather than broad claims. |

## Pilot execution docs

| File | Purpose | Use when |
| --- | --- | --- |
| `pilot_handoff_summary.md` | Plain-language summary of what the first pilot does, supports, and requires from the customer. | Use once interest is real and the team needs a shared view of the pilot shape. |
| `FIRST_LIVE_PILOT_RUNBOOK.md` | Step-by-step path from export intake through evaluation and decision review. | Use when the pilot owner asks how the first live run will actually be executed. |
| `support_v1_pilot_scorecard.md` | Simple scorecard for judging routing quality, guardrails, and operational value. | Use when defining pilot success criteria and review cadence before launch. |

## Onboarding / intake docs

| File | Purpose | Use when |
