# Support V1 Calibration Experiment Notes

The first calibration experiment tested a narrow support-specific rule for clean recent histories: if the visible window has dense recent cooperative evidence, no contradiction events, no inactivity pressure, and supportive `trust`/`cooperativeness`, then increase `overall_confidence` and decrease `unknownness` before routing. On the current 8-label slice, that changed the original result from `5/8` to `8/8` by flipping exactly the three labeled `fast_path` misses (`NW-1001`, `BO-2004`, `LO-5075`) from `deep_path` to `fast_path`. The flagged `deep_path` cases stayed unchanged, including all labels marked `contradiction_present`, `profile_too_stale`, and `wrong_first_impression`.

That is promising because it matches the main diagnosis from the first labeled run: support_v1 was not failing on obvious guardrail misses, it was failing because clean routine cases remained under-confident and over-unknown. The calibration rule improved those one-sided false negatives without weakening the current contradiction or staleness protections on this slice.

The result may still be overfit to the tiny current label set. The rule was designed after inspecting the exact failure pattern in only three clean `fast_path` examples, and the eligibility checks line up closely with those same examples: recent evidence, multiple positive/cooperative events, at least one fulfilled commitment, and no recent negatives. A perfect `8/8` on such a small hand-authored slice is therefore evidence of plausibility, not evidence that the calibration generalizes.

The next ablation experiment should answer a mechanism question: which part of this improvement is actually doing useful work? In particular, it should isolate whether the gain comes from the clean-history eligibility filter itself, the `overall_confidence` boost, the `unknownness` reduction, or only their combination, and whether any one component starts promoting stale, sparse, or contradiction-bearing histories that should remain on `deep_path`.
